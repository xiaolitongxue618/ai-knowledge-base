## 第一个坑：数据库连接问题

### 问题

`sqlite3.connect(":memory:")` 在测试模式下，每次连接都是新的空数据库，导致表不存在。

### 错误信息

```
no such table: documents
```

### 解决方案

```python
class MetadataStore:
    def __init__(self, db_path: str):
        self.is_memory_mode = (db_path == ":memory:")

        # 内存模式：保持连接打开
        if self.is_memory_mode:
            self.conn = sqlite3.connect(":memory:")
            self._init_db_cursor(self.conn.cursor())
            self.conn.commit()
        else:
            self._init_db()

    def _get_connection(self):
        if self.is_memory_mode:
            return self.conn
        else:
            return sqlite3.connect(self.db_path)
```

**关键点：**
- 内存模式必须保持连接打开
- 其他方法使用 `_get_connection()` 获取连接
- 文件模式每次新建连接

## 第二个坑：Windows 终端编码

### 问题

测试文件中的 emoji 导致 Windows 终端报错：

```
UnicodeEncodeError: 'gbk' codec can't encode character '\U0001f9ea'
```

### 解决方案

移除 emoji，使用纯文本：

```python
# 之前
print("🧪 元数据存储测试")

# 之后
print("[TEST] 元数据存储测试")
print("[OK] 创建文档成功")
```

**或者**设置终端编码为 UTF-8（更彻底的方案）。

### 2. 文档解析器

**支持格式：**
- ✅ PDF（使用 pypdf）
- ✅ Markdown（原生读取）
- ✅ 纯文本（UTF-8/GBK）

**实现细节：**
```python
def _parse_pdf(self, file_path):
    reader = pypdf.PdfReader(file_path)
    text_parts = []

    for page in reader.pages:
        page_text = page.extract_text()
        if page_text.strip():
            text_parts.append(page_text)

    return "\n\n".join(text_parts)
```

**测试结果：**
```
[OK] 文档解析成功
   文件类型: md
   文本长度: 211 字符
   标题结构: 4 个标题
```

### 3. 文本分块器

**Markdown 按结构分块：**
- 识别标题（#, ##, ###）
- 每个标题及其内容为一个块

**PDF/TXT 按字符数分块：**
- 优先在段落边界切分（\n\n）
- 其次在句子边界切分（。）
- 最后强制切分

**测试结果：**
```
[OK] 分块成功: 共 4 个块

块 1 - 9 字符
块 2 - 56 字符
块 3 - 92 字符
块 4 - 完整内容
```

## 收获与体会

### 1. 分层设计很重要

- 配置层（Settings）
- 数据层（Storage）
- 业务层（RAG Chain）
- 界面层（UI）

每层职责清晰，便于调试。

### 2. 状态机要真正落地

不是画在 PPT 上就好，代码里要：
- 先创建 indexing 状态
- 成功 → active
- 失败 → failed

### 3. 测试驱动开发

先写测试，再写代码：
- 知道要实现什么
- 即时验证
- 有回归保护

### 4. 日志很关键

开发时日志帮我快速定位问题：
```python
logger.info(f"文档解析成功: {file_path}")
logger.error(f"文档解析失败: {e}")
```
