## 第一部分：本地电脑端操作

### 1. 环境准备

#### 1.1 必需软件

**Python 环境：**
- Python 3.8 或更高版本
- 下载：https://www.python.org/downloads/

**Git 客户端：**
- Windows：https://git-scm.com/download/win
- 或使用 PowerShell 自带功能

**代码编辑器（可选）：**
- VS Code：https://code.visualstudio.com/
- PyCharm：https://www.jetbrains.com/pycharm/

#### 1.2 安装 Ollama（本地开发用）

**Windows 安装：**
1. 访问：https://ollama.com/download
2. 下载 Windows 版本
3. 安装到自定义路径（如 `D:\worksoft\Ollama`）
4. 打开命令行，验证安装：

```bash
ollama --version
```

**拉取模型：**
```bash
# Embedding 模型（用于文本向量化）
ollama pull nomic-embed-text

# LLM 模型（用于问答生成）
ollama pull qwen2.5:0.5b
```

### 2. 项目开发

#### 2.1 获取项目代码

**方式 1：从 GitHub 克隆（推荐）**
```bash
git clone https://github.com/xiaolitongxue618/ai-knowledge-base.git
cd ai-knowledge-base/ai-knowledge-base
```

**方式 2：下载 ZIP 压缩包**
1. 访问：https://github.com/xiaolitongxue618/ai-knowledge-base
2. 点击 "Code" → "Download ZIP"
3. 解压到本地目录

#### 2.2 安装项目依赖

**创建虚拟环境：**
```bash
# Windows PowerShell
cd ai-knowledge-base/ai-knowledge-base
python -m venv venv

# 激活虚拟环境
# Windows:
.\venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate
```

**安装依赖包：**
```bash
pip install -r requirements.txt
```

**依赖说明：**
- `chromadb==0.4.22` - 向量数据库
- `ollama==0.4.0` - Ollama Python 客户端
- `streamlit==1.31.0` - Web 框架
- `pypdf==3.17.4` - PDF 解析
- `pydantic>=2.9.0` - 数据验证
- 其他工具库...

#### 2.3 配置文件

**查看配置文件：** `src/config.py`

**关键配置项：**
```python
# Ollama 配置
ollama_base_url: str = "http://localhost:11434"
ollama_model: str = "qwen2.5:0.5b"          # LLM 模型
embedding_model: str = "nomic-embed-text"   # Embedding 模型

# 应用配置
chunk_size: int = 500           # 文本分块大小
chunk_overlap: int = 50         # 分块重叠
top_k_results: int = 3          # 检索文档数量
min_similarity: float = 0.5     # 最小相似度阈值

# 路径配置
data_dir: str = "./data"
vector_store_dir: str = "./data/vector_store"
metadata_db: str = "./data/metadata/knowledge_base.db"
log_file: str = "./logs/app.log"
```

**修改配置（如需要）：**
- 更改模型：修改 `ollama_model` 或 `embedding_model`
- 调整分块大小：修改 `chunk_size`
- 调整相似度阈值：修改 `min_similarity`

### 5. 文档预处理

#### 5.1 文档质量检查清单

**上传前检查：**

**内容完整性：**
- [ ] 文档内容完整，没有缺失
- [ ] 没有乱码或特殊符号
- [ ] 图片内容已转成文字
- [ ] 表格内容已转成文字或 Markdown

**格式规范性：**
- [ ] 标题层级清晰（#、##、###）
- [ ] 段落分明，有大块空白
- [ ] 列表使用规范（-、1.、*）
- [ ] 没有重复内容

**可读性：**
- [ ] 语句通顺，没有语病
- [ ] 专业术语有解释
- [ ] 没有过长的段落（建议 < 500 字）
- [ ] 关键信息明确

#### 5.2 使用预处理脚本

**运行脚本：**
```bash
python tools/preprocess_document.py input.md output.md
```

**脚本功能：**
- 移除多余空白
- 移除特殊符号
- 统一中英文引号
- 移除页眉页脚
- 质量检查报告

**质量检查报告示例：**
```
==================================================
文档质量检查报告: input.md
==================================================
✅ 内容长度合适
✅ 没有异常特殊符号
✅ 没有明显乱码
✅ 段落长度合理
==================================================
✅ 清理后的文件已保存: output.md
```

#### 5.3 推荐的文档格式

**Markdown 格式（最佳）：**
````markdown
# 文档标题

## 1. 概述

这里是概述内容...

## 2. 核心概念

### 2.1 什么是 Agent？

Agent 是一个智能代理系统...

### 2.2 核心能力

1. 感知环境
2. 决策规划
3. 任务执行

## 3. 应用场景

- 客户服务
- 数据分析
- 自动化办公
`````

**避免的格式：**
- ❌ 扫描版 PDF（图片格式）
- ❌ 大量图片内容
- ❌ 格式混乱
- ❌ 内容重复
- ❌ 段落过长
- ❌ 特殊符号多
- ❌ 有乱码

**自动备份脚本（可选）：**
```bash
# 创建备份脚本
cat > ~/backup_ai_kb.sh << 'EOF'
#!/bin/bash
BACKUP_DIR=~/backups/$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR
cp -r ~/ai-knowledge-base/data $BACKUP_DIR/
cp -r ~/ai-knowledge-base/logs $BACKUP_DIR/
echo "备份完成: $BACKUP_DIR"
EOF

chmod +x ~/backup_ai_kb.sh

# 添加到 crontab（每周日凌晨 2 点）
crontab -e

# 添加这一行：
0 2 * * 0 /home/ubuntu/backup_ai_kb.sh
```

### 6. 故障排查

#### 6.1 网站无法访问

**步骤 1：检查 PM2 进程**
```bash
pm2 status
```

**预期输出：**
```
┌────┬──────────┬─────────┬─────────┬──────────┬────────┬──────┬──────────┐
│ id │ name     │ status  │ memory  │ user     │ uptime │ ↺    │ status    │
├────┼──────────┼─────────┼─────────┼──────────┼────────┼──────┼──────────┤
│ 0  │ ai-kb    │ online  │ 3.8mb   │ ubuntu   │ 1h     │ 0    │ online    │
└────┴──────────┴─────────┴─────────┴──────────┴────────┴──────┴──────────┘
```

**如果不是 online：**
```bash
pm2 restart ai-kb
```

**步骤 2：检查 Ollama 服务**
```bash
sudo systemctl status ollama
```

**如果不是 active (running)：**
```bash
sudo systemctl start ollama
```

**步骤 3：检查防火墙**
```bash
sudo ufw status
```

**如果 8501 端口未开放：**
```bash
sudo ufw allow 8501/tcp
sudo ufw reload
```

**步骤 4：查看错误日志**
```bash
pm2 logs ai-kb --err
tail -50 ~/ai-knowledge-base/logs/streamlit.log
```

#### 6.2 问答功能异常

**检查模型是否可用：**
```bash
ollama list
```

**测试 Ollama 连接：**
```bash
curl http://localhost:11434/api/tags
```

**检查向量数据库：**
```bash
# 进入 Python
python3

# 测试导入
from src.storage.vector_store import VectorStoreManager
vs = VectorStoreManager("./data/vector_store")
print(vs.get_stats())
```

#### 6.3 内存不足

**查看内存使用：**
```bash
free -h
```

**如果内存不足（可用 < 500MB）：**

**临时解决：**
```bash
# 重启 Ollama 释放缓存
sudo systemctl restart ollama

# 重启应用
pm2 restart ai-kb
```

**长期解决：**
- 考虑升级服务器配置
- 使用更小的模型
- 定期清理日志

#### 6.4 磁盘空间不足

**查看磁盘使用：**
```bash
df -h
```

**清理日志：**
```bash
# 清理 30 天前的日志
find ~/ai-knowledge-base/logs/ -name "*.log" -mtime +30 -delete
```

**清理旧数据（谨慎）：**
```bash
# 查找大文件
du -sh ~/ai-knowledge-base/data/* | sort -h

# 删除不需要的文档（在应用中操作）
```

#### 6.5 更新代码后出错

**回滚到上一个版本：**
```bash
cd ~/ai-knowledge-base

pm2 restart ai-kb
```

**或直接重置：**
```bash
git reset --hard HEAD~1  # 回退 1 个提交
```

## 第三部分：常见问题

### 1. 本地开发问题

#### Q1: Ollama 连接失败

**错误信息：**
```
Error: connect to Ollama failed
```

**解决方案：**
1. 检查 Ollama 是否运行：
   ```bash
   ollama list
   ```
2. 如果未运行，启动 Ollama：
   ```bash
   ollama serve
   ```
3. 检查配置文件中的 `ollama_base_url`

#### Q2: 依赖安装失败

**错误信息：**
```
error: externally-managed-environment
```

**解决方案：**
```bash
# 创建虚拟环境
python -m venv venv
.\venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux/Mac

# 在虚拟环境中安装
pip install -r requirements.txt
```

#### Q3: 端口被占用

**错误信息：**
```
Address already in use
```

**解决方案：**
```bash
# Windows: 查找占用端口的进程
netstat -ano | findstr :8501

# 结束进程
taskkill /PID <进程ID> /F

# 或使用其他端口
streamlit run ui/app.py --server.port 8502
```

### 2. 服务器问题

#### Q1: Git 推送失败

**错误信息：**
```
Password authentication is not supported
```

**解决方案：**
1. 确认仓库已设为 Public（公开）
2. 或使用 Personal Access Token
3. 或使用 SSH 密钥

#### Q2: PM2 启动失败

**错误信息：**
```
PM2 Error: File ecosystem.config.json malformated
```

**解决方案：**
1. 检查配置文件格式
2. 确保是纯 JSON 格式
3. 移除注释和非标准 JSON

#### Q3: 权限问题

**错误信息：**
```
Permission denied
```

**解决方案：**
```bash
# 使用 sudo
sudo pm2 startup

# 或修改文件权限
chmod +x start_streamlit.sh
```

---

### 3. 性能问题

#### Q1: 问答速度慢

**原因分析：**
- LLM 模型推理慢（正常）
- 文档太多，检索慢
- 网络延迟

**优化建议：**
1. 减少文档数量
2. 调整 `chunk_size` 参数
3. 使用更快的模型（如果有）
4. 优化网络连接

#### Q2: 内存占用高

**正常占用：**
- Ollama: ~740MB
- Streamlit: ~50MB
- 向量数据库: ~200MB
- 总计：~1GB

**优化方案：**
1. 重启服务释放内存
2. 使用更小的模型
3. 定期清理日志

#### Q3: 磁盘占用高

**查看占用：**
```bash
du -sh ~/ai-knowledge-base/data/*
du -sh ~/ai-knowledge-base/logs/*
```

**清理方案：**
```bash
# 清理日志
find ~/ai-knowledge-base/logs/ -name "*.log" -mtime +7 -delete

# 查找大文件
du -sh ~/ai-knowledge-base/data/* | sort -h
```
