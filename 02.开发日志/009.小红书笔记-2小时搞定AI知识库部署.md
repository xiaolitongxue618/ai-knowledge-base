## 📝 部署步骤（保姆级）

### 步骤1️⃣：代码上GitHub

**本地操作：**

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/xiaolitongxue618/ai-knowledge-base.git
git push -u origin main
```

**⚠️ 注意：** 仓库要设为Public，不然服务器拉取需要密码！

---

### 步骤2️⃣：连接服务器

```bash
ssh ubuntu@你的服务器IP
```

**⚠️ 注意：**
- 输入密码时不显示字符，正常现象
- 连接成功提示符会变成 `ubuntu@VM-XXX-ubuntu:~$`

---

### 步骤3️⃣：克隆代码

```bash
cd ~
git clone https://github.com/xiaolitongxue618/ai-knowledge-base.git
cd ai-knowledge-base
```

---

### 步骤4️⃣：创建虚拟环境

**为什么要虚拟环境？**
- Ubuntu 24.04不让直接装到系统环境
- 虚拟环境隔离依赖，不会污染系统

```bash
# 先装个包
sudo apt install python3.12-venv -y

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate
```

**✅ 成功标志：** 提示符前面出现 `(venv)`

---

### 步骤5️⃣：安装依赖

```bash
pip install -r requirements.txt
```

**💥 踩坑1：** pydantic版本冲突

**解决：**
- 本地修改 `requirements.txt`
- 把 `pydantic==2.5.0` 改成 `pydantic>=2.9.0`
- 推送到GitHub，服务器上 `git pull`

**💥 踩坑2：** 编译失败

**解决：**
```bash
sudo apt install build-essential python3-dev -y
```

---

### 步骤6️⃣：安装Ollama

**一条命令搞定：**

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

**拉取模型：**

```bash
# Embedding模型（274MB）
ollama pull nomic-embed-text

# LLM模型（397MB）
ollama pull qwen2.5:0.5b
```

**为什么选 qwen2.5:0.5b？**

这是关键决策！🔑

**背景：** 服务器内存只有3.6GB

**选项对比：**
- llama3:8b → 需要 4.5-5GB → **会OOM！** ❌
- qwen2.5:0.5b → 只要 397MB → **完美！** ✅

**结果：** 总内存占用871MB，还剩1.5GB，很充裕！

---

### 步骤7️⃣：创建数据目录

```bash
mkdir -p data/metadata
mkdir -p data/vector_store
mkdir -p logs
```

---

### 步骤8️⃣：启动应用

```bash
cd ~/ai-knowledge-base
source venv/bin/activate
streamlit run ui/app.py --server.headless true --server.port 8501
```

**✅ 成功标志：**

```
Network URL: http://10.3.0.4:8501
External URL: http://43.128.97.2:8501
```

---

### 步骤9️⃣：配置防火墙

**服务器防火墙：**

```bash
sudo ufw status
# Status: inactive（没开启，不用管）
```

**腾讯云安全组（必须！）：**

1. 登录腾讯云控制台
2. 找到你的服务器
3. 安全组 → 添加规则
4. **规则：**
   - 协议端口：TCP:8501
   - 来源：0.0.0.0/0
   - 策略：允许
5. 保存

---
