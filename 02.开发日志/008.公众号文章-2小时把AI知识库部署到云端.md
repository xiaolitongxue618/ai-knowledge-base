# 小白也能搞定！2小时把AI知识库部署到云端

## 前言

前两天我做了个AI知识库系统，能在本地传文档、问问题、给答案。

做完后挺高兴，但有个问题：只能在本地电脑用，出门在外想查资料怎么办？

于是琢磨着把它部署到云服务器上，这样任何地方都能访问了。

说实话，我是个零基础小白，对服务器、Linux 这些东西一窍不通。

但咬咬牙，还是试了试。

结果，真的搞定了！只用了2小时。

今天把整个过程记录下来，给同样想尝试的朋友一个参考。

---

## 为什么想部署到云端？

本地用着挺好的，为什么还要折腾？

**几个原因：**

1. **随时访问**

2. **分享给朋友**
   - 独乐乐不如众乐乐
   - 让朋友试试，给点反馈
   - 说不定能帮到别人

3. **学习新东西**
   - 服务器部署是必备技能
   - 现在不会，以后也得学
   - 趁热打铁，一举两得

## 第一步：代码上GitHub

**问题：** 怎么把本地代码弄到服务器？

**方案：** 通过GitHub中转

1. **本地Git初始化**
   ```
   git init
   git add .
   git commit -m "Initial commit"
   ```

2. **创建GitHub仓库**
   - 登录GitHub
   - 创建新仓库：ai-knowledge-base
   - **重要：** 设为公开（Public），不然服务器拉取需要密码

3. **推送到GitHub**
   ```
   git remote add origin https://github.com/xiaolitongxue618/ai-knowledge-base.git
   git push -u origin main
   ```

   第一推会弹浏览器登录，按提示操作就行。

**踩坑：** 一开始用HTTPS认证，报错"密码认证不支持"。

**解决：** 把仓库设为Public就好了。

---

## 第二步：连接服务器

打开PowerShell，输入：

```bash
ssh ubuntu@43.128.97.2
```

**说明：**
- `ubuntu` 是用户名
- `43.128.97.2` 是服务器IP
- 输入密码时不会显示字符，正常现象

连接成功后，提示符变成这样：

```
ubuntu@VM-0-4-ubuntu:~$
```

**第一步完成！** ✅

---

## 第三步：安装环境

### 3.1 检查环境

先看看服务器上有啥：

```bash
python3 --version  # Python 3.12.3
pip3 --version     # pip 24.0
git --version      # git 2.43.0
```

**太好了！** 都已经装好了。

### 3.2 克隆代码

```bash
cd ~
git clone https://github.com/xiaolitongxue618/ai-knowledge-base.git
```

52个文件，瞬间下载完成。

### 3.3 创建虚拟环境

**踩坑1：** 直接 `pip install` 报错：

```
error: externally-managed-environment
```

**原因：** Ubuntu 24.04不让直接装到系统环境。

**解决：** 创建虚拟环境

```bash
cd ~/ai-knowledge-base
python3 -m venv venv
```

**踩坑2：** 又报错："ensurepip not available"

**解决：** 先装个包

```bash
sudo apt install python3.12-venv -y
```

然后重新创建虚拟环境，成功！

激活虚拟环境：

```bash
source venv/bin/activate
```

提示符前面出现 `(venv)`，说明激活成功了。

---

## 第四步：安装依赖

```bash
pip install -r requirements.txt
```

**踩坑3：** pydantic版本冲突

```
ERROR: ollama 0.4.0 depends on pydantic>=2.9.0
```

**原因：** 我之前写的是 `pydantic==2.5.0`，太旧了。

**解决：**
1. 本地修改 requirements.txt：改成 `pydantic>=2.9.0`
2. 提交并推送到GitHub
3. 服务器上 `git pull` 拉取更新
4. 重新安装

**踩坑4：** chroma-hnswlib 编译失败

**解决：** 安装构建工具

```bash
sudo apt install build-essential python3-dev -y
```

重新安装，成功！

**第四步完成！** ✅

---

## 第五步：安装Ollama

Ollama是运行AI模型的工具。

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

一条命令搞定，自动安装并启动为系统服务。

验证：

```bash
ollama --version
# ollama version is 0.18.2
```

**拉取模型：**

```bash
# Embedding模型（小，先拉这个）
ollama pull nomic-embed-text
# 274MB，很快

# LLM模型（大一点）
ollama pull qwen2.5:0.5b
# 397MB，也很快
```

**为什么选 qwen2.5:0.5b？**

这是关键决策！

**背景：** 服务器内存只有3.6GB，可用约2.6GB。

**选项对比：**
- llama3:8b 需要 4.5-5GB → 肯定OOM（内存溢出）
- qwen2.5:0.5b 只要 397MB → 完美适配！

**结果：** 总内存占用约871MB，还剩1.5GB，很充裕。

**第五步完成！** ✅

## 第六步：启动应用

### 6.1 创建数据目录

```bash
mkdir -p data/metadata
mkdir -p data/vector_store
mkdir -p logs
```

### 6.2 启动Streamlit

```bash
cd ~/ai-knowledge-base
source venv/bin/activate
streamlit run ui/app.py --server.headless true --server.port 8501
```

**成功！**

```
You can now view your Streamlit app in your browser.
Network URL: http://10.3.0.4:8501
External URL: http://43.128.97.2:8501
```

`43.128.97.2:8501` 就是公网访问地址！

**注意：** 当前终端被Streamlit占用了，要做其他操作需要新开SSH连接。

## 第七步：配置防火墙

**检查服务器防火墙：**

```bash
sudo ufw status
# Status: inactive
```

**好消息！** 防火墙没开，不用配置。

**但是：** 腾讯云有安全组，得在那里配置。

### 7.1 腾讯云控制台配置

1. 登录 https://console.cloud.tencent.com/
2. 进入 **云服务器CVM**
3. 找到服务器 43.128.97.2
4. 点击 **安全组** → **配置规则**
5. **添加规则：**
   - 类型：自定义
   - 来源：0.0.0.0/0（所有IP，或者指定你的IP）
   - 协议端口：TCP:8501
   - 策略：允许
6. 保存

**第七步完成！** ✅

## 遇到问题怎么办？

**我的经验：**

1. **仔细阅读错误信息**
   - 大部分错误信息都告诉你问题在哪
   - 不要慌，慢慢看

2. **善用搜索引擎**
   - StackOverflow、GitHub Issues
   - 很多问题别人都遇到过

3. **理解原理，不是死记命令**
   - 知道为什么要这样做
   - 下次遇到类似问题才能举一反三

4. **记录过程**
   - 写日志、写文章
   - 帮自己整理思路
   - 也方便以后回顾
