# PM2 进程守护部署指南

## 问题
Streamlit 进程在 SSH 断开或服务器重启后会停止，导致网站无法访问。

## 解决方案
使用 PM2 进程管理器，实现：
- ✅ 后台运行（不占用终端）
- ✅ 自动重启（崩溃后自动恢复）
- ✅ 开机自启（服务器重启后自动启动）
- ✅ 日志管理（自动记录日志）

---

## 部署步骤

### 步骤 1：安装 PM2

```bash
# 安装 Node.js 和 npm
sudo apt update
sudo apt install npm -y

# 使用 npm 安装 PM2
sudo npm install -g pm2

# 验证安装
pm2 --version
```

---

### 步骤 2：准备启动脚本

**已在项目中创建：**
- `start_streamlit.sh` - 启动脚本
- `ecosystem.config.json` - PM2 配置文件

**确保文件在服务器上：**
```bash
cd ~/ai-knowledge-base
ls -la start_streamlit.sh ecosystem.config.json
```

**如果没有（在本地执行）：**
```bash
# 提交并推送 PM2 配置到 GitHub
git add start_streamlit.sh ecosystem.config.json
git commit -m "Add PM2 process manager configuration"
git push
```

**然后在服务器上：**
```bash
cd ~/ai-knowledge-base
git pull
```

---

### 步骤 3：配置执行权限

```bash
cd ~/ai-knowledge-base
chmod +x start_streamlit.sh
```

---

### 步骤 4：停止当前运行的 Streamlit

```bash
# 查找并停止
ps aux | grep streamlit
kill <进程ID>

# 或直接杀死所有 streamlit 进程
pkill -f streamlit
```

---

### 步骤 5：使用 PM2 启动

```bash
cd ~/ai-knowledge-base

# 方式 1：使用配置文件（推荐）
pm2 start ecosystem.config.json

# 方式 2：直接启动（备选）
pm2 start start_streamlit.sh --name ai-kb
```

---

### 步骤 6：验证状态

```bash
# 查看进程状态
pm2 status

# 预期输出：
# ┌─────┬──────────┬─────────────┬─────────┬─────────┬──────────┬──────────┐
# │ id  │ name     │ status      │ cpu     │ memory  │ user     │ uptime   │
# ├─────┼──────────┼─────────────┼─────────┼─────────┼──────────┼──────────┤
# │ 0   │ ai-kb    │ online      │ 0%      │ 150MB   │ ubuntu   │ 0s       │
# └─────┴──────────┴─────────────┴─────────┴─────────┴──────────┴──────────┘
```

**查看日志：**
```bash
pm2 logs ai-kb
```

---

### 步骤 7：设置开机自启

```bash
# 保存当前进程列表
pm2 save

# 生成开机启动脚本
pm2 startup

# 复制输出的命令并执行
# 例如：sudo env PATH=$PATH:/usr/bin pm2 startup linux -u ubuntu --hp /home/ubuntu
```

---

### 步骤 8：测试自动重启

```bash
# 手动重启测试
pm2 restart ai-kb

# 应该看到进程快速重启
```

## 故障排查

### 问题 1：PM2 启动失败

**检查：**
```bash
# 查看详细错误
pm2 logs ai-kb --err

# 检查脚本权限
ls -la start_streamlit.sh

# 手动测试脚本
./start_streamlit.sh
```

---

### 问题 2：进程频繁重启

**原因：**
- 内存不足
- 代码错误导致崩溃

**解决：**
```bash
# 查看重启日志
pm2 logs ai-kb

# 增加内存限制
pm2 start ai-kb --max-memory-restart 2G
```

---

### 问题 3：开机没有自动启动

**检查：**
```bash
# 查看 PM2 启动脚本
cat /etc/systemd/system/pm2-ubuntu.service

# 手动启用
systemctl enable pm2-ubuntu
systemctl start pm2-ubuntu
```
