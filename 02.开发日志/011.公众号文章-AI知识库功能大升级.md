## 开始优化！

### 第一步：批量上传功能 🚀

**目标：** 一次选择多个文件，批量处理。

**实现过程：**

之前代码是这样的：

```python
uploaded_file = st.file_uploader(
    "选择文档",
    type=['pdf', 'md', 'txt']
)
```

改成这样：

```python
uploaded_files = st.file_uploader(
    "选择文档（支持批量）",
    type=['pdf', 'md', 'txt'],
    accept_multiple_files=True  # ← 关键参数
)
```

就这么简单！加一个参数就好了。

**然后改处理逻辑：**

从处理单个文件：

```python
if uploaded_file:
    result = process_file(uploaded_file)
```

改成遍历处理：

```python
if uploaded_files:
    # 创建进度条
    progress_bar = st.progress(0, text="准备处理...")

    for i, uploaded_file in enumerate(uploaded_files):
        # 更新进度
        progress = (i) / len(uploaded_files)
        progress_bar.progress(progress, text=f"正在处理: {uploaded_file.name}")

        # 处理文件
        result = process_file(uploaded_file)

    # 完成进度
    progress_bar.progress(1.0, text="处理完成！")

    # 显示汇总统计
    st.metric("✅ 成功", success_count)
    st.metric("❌ 失败", failed_count)
    st.metric("📄 总计", len(uploaded_files))
```

**效果对比：**

**优化前：**
- 上传 10 个文件
- 需要操作 10 次
- 总耗时：约 5 分钟（含重复操作）

**优化后：**
- 上传 10 个文件
- 只需操作 1 次
- 总耗时：约 2 分钟（纯处理时间）

**提升：效率翻倍！** ✅

```sql
    sources TEXT,        -- JSON 格式

)
```

**保存问答：**

```python
# 每次问答后保存
rag_chain.metadata_store.save_chat(
```

**显示历史：**

### 第三步：文档搜索和过滤 🔍

**目标：** 快速找到目标文档。

**实现的功能：**

1. **按名称搜索** - 模糊匹配
2. **按类型过滤** - PDF/MD/TXT
3. **按状态过滤** - 活跃/失败
4. **排序** - 时间/大小/名称

**界面设计：**

```python
# 搜索框
search_query = st.text_input("🔎 搜索文档名")

# 过滤器
filter_type = st.selectbox("📁 文档类型", ["全部", "pdf", "md", "txt"])
filter_status = st.selectbox("📊 状态", ["全部", "active", "failed"])
sort_by = st.selectbox("🔃 排序", ["时间", "大小", "名称"])
```

**过滤逻辑：**

```python
filtered_docs = docs.copy()

# 搜索过滤
if search_query:
    filtered_docs = [d for d in filtered_docs
                     if search_query.lower() in d.file_name.lower()]

# 类型过滤
if filter_type != "全部":
    filtered_docs = [d for d in filtered_docs
                     if d.file_type == filter_type]

# 状态过滤
if filter_status != "全部":
    filtered_docs = [d for d in filtered_docs
                     if d.status.value == filter_status]

# 排序
if sort_by == "大小":
    filtered_docs = sorted(filtered_docs, key=lambda x: x.file_size, reverse=True)
elif sort_by == "名称":
    filtered_docs = sorted(filtered_docs, key=lambda x: x.file_name)
```

**效果：**
- ✅ 输入关键词立即筛选
- ✅ 组合过滤条件
- ✅ 灵活排序
- ✅ 显示结果数量

**使用场景：**

想找所有 PDF 文档：
- 类型选择"pdf"
- 立即显示所有 PDF

想找包含"Python"的文档：
- 搜索框输入"Python"
- 立即显示匹配文档

想找最大的文档：
- 排序选择"大小"
- 文档按大小降序排列

## 踩过的坑

### 坑 1：expander() 不支持 key 参数

**错误代码：**

```python
with st.expander("标题", key="xxx"):  # ← 报错
```

**错误信息：**
```
TypeError: expander() got an unexpected keyword argument 'key'
```

**原因：** Streamlit 的 `expander()` 组件不支持 `key` 参数（其他组件都支持）。

**解决：**

```python
with st.expander("标题"):  # ← 移除 key 参数
```

**经验：** 不同组件的参数支持不同，遇到错误先查文档。
