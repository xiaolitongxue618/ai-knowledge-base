# 从零到一构建 AI 知识库：一个零基础的完整记录

## 前言

前两天心血来潮，想做个 AI 知识库系统。

想法很简单：上传几个 PDF 文档，然后能像 ChatGPT 那样跟它对话，让它基于文档内容回答问题。

听上去不难吧？但作为一个零基础的编程小白，整个过程还是踩了不少坑。

今天把整个过程记录下来，给同样想学习 AI 的朋友一个参考。

平常看技术文档、学习资料，经常遇到这些问题：

## 第一步：理解核心原理

在动手之前，我先花时间搞懂了 RAG 的原理。

**RAG 是什么？**

全称是 Retrieval-Augmented Generation（检索增强生成）。

说白了就三步：

1. **检索**：在文档里找相关内容
2. **增强**：把找到的内容整理一下
3. **生成**：让 AI 基于这些内容回答问题

**为什么要这样做？**

因为普通的 LLM（比如 GPT）有两个问题：
- 训练数据有截止日期，新知识不知道
- 容易"幻觉"，瞎编乱造

RAG 通过给 LLM 提供参考文档，让它"有据可依"，就能解决这些问题。

## 第三步：从最小功能开始

没有一上来就想做完美，而是先做一个能用的版本。

### 3.1 元数据存储

最简单的问题：怎么记录上传了哪些文档？

用 SQLite 建了个表：

```sql
CREATE TABLE documents (
    id INTEGER PRIMARY KEY,
    source_id TEXT UNIQUE,  -- 文件的 MD5 哈希
    file_name TEXT,
    file_type TEXT,
    status TEXT,             -- indexing/active/failed
    upload_time TIMESTAMP
);
```

**为什么要 MD5？**

为了避免重复上传。同一个文件，MD5 肯定一样，查一下就知道有没有存过。

**踩坑 1：SQLite 内存模式**

测试时用 `:memory:` 模式，结果发现表不存在。

原因是：内存模式下，每次新建连接都是空数据库。

解决：内存模式要保持连接打开。

### 3.2 文档解析

支持三种格式：

**PDF：** 用 pypdf 库
```python
import pypdf
reader = pypdf.PdfReader("file.pdf")
text = ""
for page in reader.pages:
    text += page.extract_text()
```

**Markdown：** 直接读文件，顺便提取标题结构

**TXT：** 简单，但要考虑编码问题（UTF-8/GBK）

### 3.3 文本分块

这个很关键。

**为什么要分块？**

LLM 有上下文长度限制，一个 100 页的 PDF 肯定塞不进去。

**怎么分？**

- Markdown：按标题分块（每个 #、## 是一个块）
- PDF/TXT：按字符数分块（500 字符），优先在段落边界切

**为什么要重叠？**

假设 chunk_size=500，overlap=50：

- 第 1 块：0-500
- 第 2 块：450-950（重叠 50）
- 第 3 块：900-1400

这样可以避免在句尾切分，丢失上下文。

### 3.4 向量化

分块之后，要把每个文本块转成向量（一串数字）。

**用什么模型？**

一开始想用 OpenAI 的 embedding，但：
- 要钱
- 数据要传到云端

后来发现 Ollama 有个 nomic-embed-text 模型：
- 免费
- 本地运行
- 效果不错

**踩坑 2：向量相似度计算**

一开始用 ChromaDB 的默认设置（L2 距离），结果距离值都是几十到几百，相似度全是负数。

问题是：L2 距离不适合 768 维的文本向量。

解决：改用余弦距离，并修正相似度计算：
```python
# 余弦距离范围是 0-2
# 相似度 = 1 - (距离 / 2)
similarity = 1 - (distance / 2)
```

修改后，相似度正常了，检索质量也好很多。

### 3.5 向量存储

用 ChromaDB 存向量：

```python
collection.add(
    documents=[文本列表],
    embeddings=[向量列表],
    metadatas=[元数据列表],
    ids=[唯一ID列表]
)
```

检索时：
```python
results = collection.query(
    query_embeddings=[问题向量],
    n_results=3  # 返回最相关的 3 个
)
```

### 3.6 LLM 生成答案

检索到相关文档后，让 LLM 基于这些内容回答：

```python
prompt = f"""
参考信息：
{检索到的文档}

用户问题：{问题}

请基于参考信息回答问题。
"""

answer = ollama.generate(prompt)
```

## 第五步：测试与优化

### 测试过程

**单元测试：**
- 元数据存储测试
- 文档解析测试

**集成测试：**
- 端到端文档上传流程
- 完整问答流程

### 发现的问题

**问题 1：Windows 终端编码**

测试文件用了 emoji，Windows 终端报错。

解决：全部改成纯文本标记 `[OK]`、`[ERROR]`。

**问题 2：向量检索阈值**

一开始阈值设 0.3，结果什么都检索不到。

原因：相似度计算方式改了，阈值也要改。

解决：改成 0.5（余弦距离下，0.5 算中等相关）。
