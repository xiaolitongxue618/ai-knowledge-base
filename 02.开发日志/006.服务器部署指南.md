# 服务器部署指南 - 腾讯云新加坡

## 🎯 部署目标

将 AI 知识库系统部署到腾讯云新加坡服务器，实现公网访问。

- [x] SSH 客户端（PowerShell/CMD 自带）

## 步骤 1: 连接服务器

### 1.1 获取服务器信息

登录腾讯云控制台，查看：
- 服务器公网 IP：`xxx.xxx.xxx.xxx`
- 登录密码/密钥

### 1.2 使用 SSH 连接

**在 PowerShell 或 CMD 中运行：**

```bash
ssh root@你的服务器IP
```

**例如：**
```bash
ssh root@123.45.67.89
```

**输入密码**（不会显示字符，直接输入回车）

连接成功后，你会看到类似：
```
Welcome to Tencent Cloud!
[root@VM-XXXX-XXX ~]#
```

### 1.3 验证系统

```bash
# 查看系统版本
cat /etc/os-release

# 查看内存
free -h

# 查看磁盘
df -h
```

**预期结果：**
- 内存：约 4GB
- 磁盘：约 60GB
- 系统：CentOS/Ubuntu

## 步骤 4: 拉取 AI 模型

### 4.1 配置模型存储路径（可选）

**如果不想把模型存在 C 盘：**

```bash
# 创建模型存储目录
mkdir -p /worksoft/Ollama

# 设置环境变量
echo 'export OLLAMA_MODELS=/worksoft/Ollama' >> ~/.bashrc
source ~/.bashrc
```

### 4.2 拉取模型

**拉取 Embedding 模型（先拉这个，小）：**
```bash
ollama pull nomic-embed-text
```

**拉取 LLM 模型（大，需要时间）：**
```bash
ollama pull llama3:8b
```

### 4.3 验证模型

```bash
ollama list
```

**预期输出：**
```
NAME                      ID              SIZE      MODIFIED
nomic-embed-text:latest   ...            274MB     ...
llama3:8b                 ...            4.7GB     ...
```

## 步骤 5: 上传代码

### 5.1 方法 A：使用 Git（推荐）

**在本地电脑（项目目录）：**

```bash
# 初始化 Git 仓库
git init
git add .
git commit -m "Initial commit: AI Knowledge Base"

# 推送到 GitHub/Gitee
# 然后在服务器上克隆
```

**在服务器上：**
```bash
# 克隆代码（替换为你的仓库地址）
git clone https://github.com/你的用户名/ai-knowledge-base.git
cd ai-knowledge-base/ai-knowledge-base
```

### 5.2 方法 B：手动上传（如果没有 Git）

**在本地电脑压缩项目：**

```bash
# 压缩项目（排除不必要的文件）
tar -czf ai-knowledge-base.tar.gz \
  --exclude='.git' \
  --exclude='__pycache__' \
  --exclude='*.pyc' \
  --exclude='data/*' \
  --exclude='logs/*' \
  ai-knowledge-base/
```

**使用 SCP 上传：**
```bash
scp ai-knowledge-base.tar.gz root@你的服务器IP:/root/
```

**在服务器上解压：**
```bash
cd /root
tar -xzf ai-knowledge-base.tar.gz
```

## 步骤 10: 配置防火墙

### 10.1 腾讯云安全组配置

**登录腾讯云控制台：**

1. 找到你的服务器实例
2. 点击"安全组"
3. 添加规则：

```
类型：自定义
来源：0.0.0.0/0（或指定 IP 段）
协议端口：TCP:8501
策略：允许
```

4. 保存

### 10.2 服务器防火墙（如果启用）

**Ubuntu/Debian:**
```bash
ufw allow 8501/tcp
ufw reload
```

**CentOS:**
```bash
firewall-cmd --permanent --add-port=8501/tcp
firewall-cmd --reload
```

## 🔧 进阶配置

### 使用进程管理器（推荐）

**安装 PM2：**
```bash
npm install -g pm2
```

**创建启动脚本：**

```bash
cat > start_ollama.sh << 'EOF'
#!/bin/bash
ollama serve
EOF

chmod +x start_ollama.sh
```

**启动服务：**

```bash
# Ollama 服务
pm2 start start_ollama.sh --name ollama

# Streamlit 应用
pm2 start "streamlit run ui/app.py --server.headless true" --name ai-kb
```

**查看状态：**
```bash
pm2 status
pm2 logs ai-kb
```

### 设置开机自启

```bash
pm2 startup
pm2 save
```

## ❗ 常见问题

### 问题 1：端口被占用

**解决：**
```bash
# 查看占用 8501 端口的进程
lsof -i:8501

# 杀死进程
kill -9 进程ID
```

### 问题 2：Ollama 连接失败

**检查：**
```bash
# 查看 Ollama 是否运行
ps aux | grep ollama

# 重启 Ollama
pm2 restart ollama
```

### 问题 3：访问很慢

**可能原因：**
- 网络延迟（新加坡 → 中国）
- 服务器负载高
- LLM 生成慢（正常）

**优化：**
- 上传文档控制在 10MB 以内
- 调整 chunk_size 减少块数
- 使用更快的模型（如果可用）
